# BUG-004 - Campo Nome permite/incentiva preenchimento de data sem validação adequada

## Módulo
Novo ciclo

## Severidade
Média

## Prioridade
Média

## Título
Campo Nome do novo ciclo apresenta exemplo de data/período sem validação adequada para o preenchimento.

## Passos para Reprodução
1. Acessar a funcionalidade **Novo ciclo**.
2. Localizar o campo **Nome**.
3. Observar o texto de exemplo exibido no campo.
4. Informar um valor seguindo o formato sugerido ou inserir dados inválidos relacionados ao período.

## Resultado Atual
O campo **Nome** apresenta como exemplo:

"Ex.: Avaliação Q2/2026"

Porém, não existe uma validação clara ou orientação sobre o formato esperado para o preenchimento do campo, permitindo possíveis inconsistências no cadastro do ciclo.

## Resultado Esperado
O sistema deve apresentar uma orientação clara sobre o formato esperado para o nome do ciclo e validar o preenchimento conforme a regra definida.

Exemplo:
- "Avaliação 2º trimestre 2026"
- "Ciclo de Avaliação Q2/2026"

## Impacto
Usuários podem cadastrar ciclos com nomes fora do padrão esperado, gerando inconsistência nas informações e dificultando a organização e identificação dos ciclos cadastrados.
